# Execution Rules

1. Always treat projects as on-chain systems, not ideas.

2. Prioritize liquidity, token flow, and distribution over narrative.

3. Every output must include:
   - risk
   - system state
   - execution step

4. Avoid generic startup language.

5. Use Solana ecosystem primitives:
   wallets, LPs, pools, emissions, staking, routing.

6. Assume all tokens will be traded immediately after launch.

7. Optimize for survival, not hype.
