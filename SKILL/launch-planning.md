Launch Planning Skill

Purpose

Design a structured, execution-focused launch plan for Solana projects across pre-launch, launch, and post-launch phases.

This skill converts vague ideas into a clear execution roadmap with timelines, deliverables, and KPIs.

⸻

When to use this skill

Use when the user asks about:

* Launching a Solana project
* Building go-to-market strategy
* Pre-TGE planning
* Product launch execution
* Growth timelines

⸻

Input Required

* Project name
* Stage (idea / building / testnet / pre-launch / live)
* Budget (optional)
* Team size (optional)
* Target users or niche

⸻

Output Structure

1. Launch Phases

Phase 1: Foundation (Week 1–2)

* Branding and positioning
* Landing page + waitlist
* Social setup (X, Discord, Telegram)
* Core messaging definition

KPIs:

* 500–2,000 waitlist users
* 1 clear value proposition

⸻

Phase 2: Pre-Launch Growth (Week 3–6)

* Content engine on X
* Community engagement loops
* Ambassador recruitment
* Early partnerships

KPIs:

* 5,000–15,000 community members
* 30–50% engagement rate (Discord/X)

⸻

Phase 3: Hype & Distribution (Week 7–10)

* Airdrop campaign
* KOL onboarding
* Referral system
* Listing preparation

KPIs:

* Viral coefficient > 1.2
* 20,000+ users

⸻

Phase 4: Launch Execution (Week 11–12)

* Token or product launch
* Liquidity planning
* Post-launch retention strategy

KPIs:

* Stable retention > 40%
* Liquidity secured

⸻

Strategic Principles

* Distribution > Product perfection
* Community before token
* Narrative before mechanics
* Execution speed matters more than complexity

⸻

Output Format Rule

Always return:

1. Timeline
2. Weekly breakdown
3. KPIs
4. Risks
5. Immediate next 3 actions
