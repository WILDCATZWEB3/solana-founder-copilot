Launch Readiness Audit Skill

Purpose

Evaluate whether a Solana project is actually ready to launch. This is a hard reality-check system that identifies missing execution layers before TGE or product release.

⸻

When to use

Use when user asks about:

* “Am I ready to launch?”
* Pre-TGE evaluation
* Product readiness
* Go-to-market validation
* Launch risk assessment

⸻

Input Required

* Project name
* Product status (idea / MVP / testnet / live)
* Community size
* Treasury size
* Partnerships (if any)
* Token status (if applicable)

⸻

Evaluation Framework

1. Product Readiness (0–25)

Check:

* Working MVP exists
* Onboarding flow is clear
* Core feature is functional
* UX is usable without founder assistance

⸻

2. Community Strength (0–25)

Check:

* Active users vs total members
* Engagement rate
* Content activity
* Retention signals

⸻

3. Distribution System (0–20)

Check:

* X growth engine
* Referral loops
* KOL strategy
* Ambassador program

⸻

4. Treasury Stability (0–15)

Check:

* Runway duration
* Burn rate efficiency
* Stablecoin allocation
* Risk exposure

⸻

5. Ecosystem Positioning (0–15)

Check:

* Solana ecosystem alignment
* Partnerships
* Narrative strength
* Competitive positioning

⸻

Output Format

Launch Score (0–100)

* Product
* Community
* Distribution
* Treasury
* Ecosystem fit

⸻

Critical Gaps

List top 3 blockers preventing launch.

⸻

Risk Level

* Safe to launch
* Moderate risk
* High risk (not ready)

⸻

Recommendation

Either:

* Launch now with fixes
* Delay launch 2–4 weeks
* Rebuild core system
